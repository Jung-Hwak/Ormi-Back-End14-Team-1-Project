# 1차 프로젝트 1팀 회의록

- 날짜: 2025년 12월 22일
- 참여자: 정확 (팀장), 정상원, 김영호

## 오전 회의 내용

> [!IMPORTANT]
>
> - 로그인 페이지에서 소셜 로그인을 누르면 새로운 팝업창이 뜨게 만드는게 더 나을거 같다.
> - 유저 친화적(사용에 대한 유저들의 편의성)을 높여 줄만한 요소를 고민해보자
> - 소셜 로그인 버튼에 각각으 로고를 넣어보자
> - 로그인활성화 유지 하는 쿠키를 자바스크립트로 구현을 어떻게 할지 래퍼런스 확인하기



## 작업 목표

> [!IMPORTANT]
>
> - 정확 : 로그인 페이지 스타일 완성하기, 로그인 페이지의 아이디 비밀번호 확인하는 기능, 회원가입 버튼 클릭시 회원가입 페이지로 연결, 소셜로그인 팝업창 만들어보기
> - 정상원 : 로그인되어있을때 로그인 버튼을 누르면 회원정보, 로그아웃 버튼이 있는 창 만들어보기, 설정 창에 도움말, 고객센터 만들어보기
> - 김영호 : 상단 네비게이션 바 통일성 있게 수정하기, 비밀번호 확인란에서 비밀번호가 틀렸는지 아닌지 확인하게 만들기, 전반적인 사용감 개선하기

## 작업 과정

> [!IMPORTANT]
>
> - 정확 : 소셜 버튼을 앵커요소에 버튼을 자식요소로 추가하거나 자바 스크립트로 다른 페이지로 이동 시키려 하였는데 강사님의 말씀을 듣고 앵커에 버튼 요소가 들어가는게 말이 이상한
>         작업 이라는 말을 듣고 앵커 요소로 #(헤시태그)와 자바 스크립트에 prevetDeflut 를 이용해 기본 이벤트를 제거하고 팝업창으로 띄워지게끔 자바 스크립트를 작성함
> - 정상원 : 로그인 후 프로필 파일 작업,  설정창에 도움말과 고객센터 작업
> - 김영호 : 비밀번호 필드에 정규식을 적용하여 영문자와 숫자만 입력되도록 강제함. 초기에는 잘못된 입력을 즉시 삭제했으나, 사용자 경험을 위해 실시간으로 피드백을 주는 방식으로 개선.
>           인증번호 4자리 입력 시 자동으로 "인증 완료" 메시지를 띄우고 입력창을 비활성화하여 완료 상태임을 명확히 함.


## 작업 회고

> [!IMPORTANT]
>
> - 정확 : 소셜 로그인을 하는 것을 어떤 태그를 가지고 만들어야 할지 고민 강사님께 여쭤본후
> - 정상원 : 프로필 파일 만드는게 생각보다 어려웠다.
> - 김영호 : 첫번째 만들었던 페이지(이용약관, 본인인증, 회원 정보 입력)의 로고 부분은 통일성 있게 수정.회원 입력 정보 수정 페이지의 자바 스크립트 파일과 HTML 스크립트 제작. 
>           이메일 수정, 비밀번호 수정, 닉네임 기능 제작.

## 내일 목표

> [!IMPORTANT]
>
> - 정확 - 자바 스크립트의 쿠키를 이용해 로그인 페이지에서 로그인을 한 이후 로그인상태 지속된 형태만들어 보기
> - 정상원 - 지금까지 만든거 다듬고 결과창 좀 더 손보기
> - 김영호 - 회원 입력 정보 페이지의 css 제작. 단, 이전에 제작했던 페이지와 달리 햄버거 아이콘, 유저 아이콘 작동되는 방식으로 제작할 것.
