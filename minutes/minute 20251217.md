- 날짜: 2025년 12월 17일
- 참여자: 정확 (팀장), 정상원, 김영호

## 오전 회의 내용

> [!IMPORTANT]
>
> - 현재까지의 작업물들을 병합 해본 결과 javascript의 작동에 오류가 생김
> - 기본적으로 가져갈 공통된 스타일을 안정해놨었기에 각자의 스타일 시트를 작성한 결과 작업물의 결과가 달라짐(공통된 스타일 시트를 써야함)
> - 공통 스타일을 가지는 메뉴바(네비게이션바) HTML 요소를 작성해야함.
> - 메인 페이지와 결과 페이지 쪽을 매끄럽게 연결이 되어야 하기에 서로의 스타일 시트 JavaScript HTML의 요소 통일



## 작업 목표

> [!IMPORTANT]
>
> - Figma 캔버스로  본인이 배정 받은 AI portal site 스캐치 하기
> - 메인 페이지와 결과 페이지의 연결을 위해 서로간의 HTML 문서 작성과 스타일 시트 작성을 통일 시키기- (정확), (정상원)
> - 사이드 바에 들어갈 설정 창 만들기 (정확)
> - 약관 페이지, JavaScript 헤드에 작성(JavaScript외부 파일 만들어 적용시키기) (김영호)

## 작업 과정

> [!IMPORTANT]
>
> - 설정창이 검색하는 텍스트 에어리어에 의해 가려져서 z-idex로 최상단으로 배치해서 잘 보이게끔 해서 해결
> - 자바 스크립트를 이용해, 계약약관 페이지에서 클릭해서 메인 페이지로 이동하는 것.
>   마찬가지로 계약약관의 동의를 받고, 다음을 클릭하면 본인인증 페이지로 이동하는 것.
>   체크 박스 + 약관 동의 확인 함수를 외부 자바 스크립트로 이동
>   브라우저 경고창이 사용자의 불쾌감을 유발 할 수 있다는 판단 하에 모달 창으로 수정. (김영호)
> - 수정된 css파일로 생기는 오류 수정.수정 후 생기는 전송이 두번씩 전송되는 오류 해결 수정 후 생기는 대화창 넓이 문제 해결 (정상원)

## 작업 회고

> [!IMPORTANT]
>
> - 정확 : 팀장으로써 역할을 제대로 이해하지 못해 팀원들과 거의 박치기 식으로 진행한 결과 현재 진행된 단계에서 작업물을 합쳐본결과 작성된 스타일시트와 스크립트가 연결이 매끄럽게 되지 못하였다. 좀 더 팀원들과 소통을 하고 생각을 하고 팀원들과 진행 단계를 세세하게 나누는 생각을 해야겠다.
> - 정상원 : 수정된 css파일로 생기는 오류 수정하는 부분에서 몇몇개 막히는 부분이 있어서 시간이 걸려서 어려웠던 부분이 크다.
> - 김영호 : 자바 스크립트를 따로 작성하는 과정에서 제대로 적용되지 않은 부분(예시: 약관 동의, 모달 창 등)
