# 1차 프로젝트 1팀 회의록

- 날짜: 2025년 12월 23일
- 참여자: 정확 (팀장), 정상원, 김영호

## 오전 회의 내용

> [!IMPORTANT]
>
> - 자바스크립트 쿠키를 이용하여 로그인시 로그인 상태가 유지되어있는 페이지만들어보자
> - 다크 모드의 바디의 배경색을 좀더 밝게 하고 네비게이션 바와 메뉴 바 쪽의 배경색을 다크모드의 바디의 배경색 보다 좀더 밝게 만들어 유저의 편의성을 높이기
> - 테마 설정 유지하도록 자바 스크립트의 localStorage 만들기 로그인 상태유지 하는 자바스크립트 localStorage 만들기



## 작업 목표

> [!IMPORTANT]
>
> - 정확 : 테마의 다크모드 라이트모드 변경만들기, 테마 변경유지 자바스크립트 작성, 로그인 상태 유지 자바스크립트 작성하기
> - 정상원 : 사라진 도움말,고객센터 다시 만들고 수정, 사이드바 채팅내역 수정
> - 김영호 : 상단 네비게이션 바 통일성 있게 수정하기, 비밀번호 확인란에서 비밀번호가 틀렸는지 아닌지 확인하게 만들기, 전반적인 사용감 개선하기

## 작업 과정

> [!IMPORTANT]
>
> - 정확 : 셋팅의 설정중에 테마 설정인 다크, 라이트 모드의 스타일 시트를 작성하고 로고, 텍스트, 이미지등의 가시성이 떠러지는걸 css의 filter를 추가하여 가시성을 높였고
>         브라우저의 localStorage를 사용하여 사용자의 테마 선택을 저장하도록 자바스크립트를 작성하였습니다.
> - 정상원 : 사라진 도움말,고객센터 다시 만들고 수정, 사이드바 채팅내역 수정
> - 김영호 : 사실상 무언가 추가했다, 뺐다를 반복하느라 시간 허비가 심했다. 깔끔하게 css 작성만 끝냈다면 더 빨랐을 거 같다. 현재 회원 입력 정보 수정란 css를 완성 시키고,
>           이전 작업과 마찬가지로 최대한 메인 페이지와 이질감 없이 작업했다.


## 작업 회고

> [!IMPORTANT]
>
> - 정확 : 소셜 로그인을 하는 것을 어떤 태그를 가지고 만들어야 할지 고민 강사님께 여쭤본후
> - 정상원 : 채팅내역 저장하게 하는 기능이 어려웠다.
> - 김영호 : 전반적인 회원 입력 정보 페이지의 css 제작 완성. 생각보다 시간이 오래 걸렸음.사이트를 둘러 보며 어떻게 작동되는 지 확인하는 중. \
>           최대한 유저에게 편한 곳을 조사하여 사용감을 어떻게 계선 할지 고민. 햄버거 아이콘과 유저 아이콘은 넣어 봤다가, 너무 별로라서 제외. 
>           사유(회원 정보 수정하다가 ai 채팅 내역을 볼 필요가 없다고 느낌 + 유저 아이콘은 누르면 바로 회원 입력 정보 수정으로 넘어가서)


## 내일 목표

> [!IMPORTANT]
>
> - 정확 - 자바 스크립트의 localStorage를 이용해 다크 모드 라이트 모드의 테마 변경 유지하기, 로그인 페이지에서 로그인을 한 이후 로그인상태 지속된 형태만들어 보기
> - 정상원 - 채팅 방 만들기
> - 김영호 - 본인 인증 페이지의 기능 추가(핸드폰, 아이핀, 카드, 토스) 등의 인증 수단을 누르면 바로 인증 완료가 아닌 모달 창이나, 팝업 창을 띄워 무언가(아무거나) 
>           입력하게 한 후 인증 완료 될 수 있도록. 회원 비밀번호 찾기 페이지 구성해보기
