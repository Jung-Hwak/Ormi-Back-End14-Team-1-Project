## 작업 목표

> [!IMPORTANT]
>
> - Figma 캔버스로  본인이 배정 받은 AI portal site 스캐치 하기
> - 정확 : 불필요한 id,class등을 지우고 그에 맞게 스타일 시트 재정립, JavaScript 재정립

>           회원 가입 페이지 프레임 만들기

## 작업 과정

> [!IMPORTANT]
>
> - 정상원 : index.html이랑 호환되게 함.그리고 결과 페이지 질문하고 답 구상
> - 정확 : 메인 페이지인 index.html에 요소에 붙여둔  id, class 선택자들을 지워내니 스타일 시트에 작성해놓은 대부분의 스타일이 적용이 되지않아.
>         인터넷 브라우저의 개발자 도구로 확인해가면서 새롭게 지정해놓은 id와 클래스로 탐색범위를 최대한 좁게 지정하려고 하며 원래 보였던 스타일대로 보이게끔 만들었음.
>         자바 스크립트만 프로그램에서 참조할 요소를 탐색하는 변수 선언등 여러 부분에서 다시 손봐야 해서 오래 걸릴거같음
> - 김영호 : 본인인증 페이지의 중앙 로고를 클릭시 메인 페이지로 이동하고, 취소시 이전 화면은 약관동의 화면으로 넘아가게 설정. 
>           스크립트 영역을 자바 스크립트로 작성해 외부 영역으로 이동.


## 작업 회고

> [!IMPORTANT]
>
> - 정확 : 선생님의 말씀을 듣고 불필요한 id,class 선택자들을 지워내고 스타일 시트의 선택자를 좀더 브라우저에 부담이 안가는 방법으로 해보기위해 오전 오후의 대부분의
>         시간을 들여 겨우 해냈고 수업이 끝나고 자바 스크립트 선택자와 탐색 요소등을 다시 재지정 해야할거 같다 그리고 현재 작업 진행형 중에서 메인 페이지와 직접적으로 연관이 있는
>         정상원 팀원 분에게 바뀐 스타일 시트와 HTML을 보여주면서 괜찮은지 문제가 생기는 부분이 있으면 말해서 조율을 하자고 하였지만 팀장으로써 부족한 부분이 많아 죄송할 따름이다.
> - 정상원 : 변경된 index.html이랑 충돌이 없게끔 하는게 어려웠다
> - 김영호 : 본인인증 페이지의 중앙 로고를 클릭시 메인 페이지로 이동하고, 취소시 이전 화면은 약관동의 화면으로 넘아가게 설정.
>   스크립트 영역을 자바 스크립트로 작성해 외부 영역으로 이동.
>   본격적인 회원가입 영역(아이디, 비밀번호, 이메일 인증 등)의 전반적인 뼈대를 작성. 아이디, 이메일 부분을 중복 확인 설정이 무척이나 어렵게 느껴진다.
