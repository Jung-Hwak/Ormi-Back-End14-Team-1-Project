# 1차 프로젝트 1팀 회의록

- 날짜: 2025년 12월 16일
- 참여자: 정확 (팀장), 정상원, 김영호

## 오전 회의 내용

> [!IMPORTANT]
>
> - 결과 창에 나오는 대답을 나오게하는 JavaScript 작성(정상원)
> - 약관 페이지의 HTML JavaScript 작성 (김영호)
> - 메인 페이지의 JavaScript 작성 (정확)
> - 네비 바에 언어 설정을 추가 해보자 (정확)
> - 다크/라이트 모드를 어디에 넣어둘지 (설정 창에 넣어두기)

## 작업 목표

> [!IMPORTANT]
>
> - 메인 사이트, 로그인 페이지, 최상단 네이게이션, 메뉴 바 버튼 활성화/비활성화(JavaScript) - (정확) 
> - 결과 페이지, 검색폼에 검색을 하면 결과 페이지로 넘어가져서 대답이 돌아오는것, 검색을 마우스클릭으로만 하는게 아니 엔터키로도 작동되게 만들기- (정상원)
> - 회원 정보 수정, 약관 페이지 - (김영호)

## 작업 과정

> [!IMPORTANT]
>
> - 정상원 : 메인 페이지에서 검색폼에 질문을 입력하면 결과페이지로 넘어가면서 결과 페이지에 대답이 나오게끔 만들고 검색폼에 마우스 버튼클릭이아닌 엔터키로도 되게끔 JavaScript를 작성해 놨음
> - 김영호 : 회원가입(약관) 페이지의 약관 동의 페이지 체크박스 와 약관 설명을 작성완료, 본인인증 페이지의 휴대폰,아이핀,토스 인증 등을 표현한 HTML 문서와 css 스타일 시트를 작성
> - 정확 : ai포털 사이트의 공통 스타일 시트 (common.css)를 만들어 보았고, 메인 페이지의 각각의 배치되는 것과 스타일을 적용할 스타일 시트를 따로 만들어 보았음
> - JavaScript로 메뉴바의 작동을 하게될 이벤트 핸들러를 작성하고 그안에서 유효성 검사등을 통해 제대로 작동하는지 확인완료.


## 작업 회고

> [!IMPORTANT]
>
> - 정상원 : JavaScript의 엔터키 이벤트를 적용 시키는게 작업이 오래 걸렸지만 제대로 작동이 되는것을 확인하였고 자신의 작업물이 다른 팀원들의 작업물과 연결이 매끄럽게 되는지 의문이큼
> - 정확 : 공통으로 작동이 될 메뉴(햄버거)버튼을 JavaScript 작성하는게 오래걸렸고 무엇보다 내가 작업한 작업물이 팀원들이 만든 작업물과 나의 작업물이 매끄럽게 연결되는지 걱정이큼
> - 다 끝날때가 되어서 팀원들과 제대로 규칙 소통을 못한거 같다는 생각이 들어서 팀장이 처음이기도 하기에 좀 더 적극적으로 팀원들과 소통을 해야하는 필요성을 마지막 회고시간에 듬
> - 김영호 : 처음으로 팀원들과 공동 작업을 해서 본인의 작업물이 잘되었다고 해도 연결이 어려울것 같은 걱정이 든다.
> - 모두가 다 공동 작업은 처음이라 현재 본인의 영역의 작업은 잘어가고 있지만 이것들을 연결시키는 문제에서 어떻게 될지 의문이라 걱정이됨.


## 내일 목표

> [!IMPORTANT]
>
> - 정확 - 오전에 로그인 페이지를 만들어보고 팀원들과 작업물을 합쳣을때 작동이 되는지 확인해보고(높은 확률로 정상적으로 안될거 같지만)정상적으로 작동이 안되면 팀원들과 다시 규칙   (스타일 시트, id class 식별자, 들어가야되는 요소 nav main asiede 등등)을 재정립을 해야됨
> - 정상원 - 대화가 쌓여서 대화창이 아래로 내려가지는 스크롤바를 생성, 첫 대화한 대화 내역이 옆의 메뉴(사이드)바에 히스토리가 남게 만들어보는 것.
> - 김영호 - 첫 번째 본인인증 옆에 붙어있는 이모지를 찾아서 만들어보는것 메뉴바 활성화 시키기, 회원가입 페이지 만들어 보기

* 공통화된 스타일들은 스타일 시트를 따로 만들어두고
* 각각의 페이지마다 필요한 스타일은 따로 작성 해두기
