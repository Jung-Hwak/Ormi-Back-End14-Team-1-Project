## 오전 회의 내용

> [!IMPORTANT]
>
> - 회원 정보 관리(수정)페이지로 연결이 되게끔 만들기

> - 채팅(질문) 히스토리 내역 만들기
> - 본인인증 페이지의 인증 클릭하면 팝업창 나오게 만들기
> - figma로 id,pw 찾기 페이지 스케치 완성하기
> - 아이디/비밀번호 찾는 페이지 만들고 아이디나, 비밀번호 찾기를 클릭하면 그 버튼에 맞는 팝업창 띄우기



## 작업 목표

> [!IMPORTANT]
>
> - 정확 : 회원 정보 관리(수정)페이지로 연결이 되게끔 만들기, 테마 설정 적용시 다른 페이지에서도 테마설정이 유지되게 하기
> - 정상원 : 결과 페이지에 질문응답 나오게 만들기, 채팅(질문) 히스토리 내역 만들기
> - 김영호 : 본인인증 페이지의 인증 클릭하면 팝업창 나오게 만들기, figma로 id,pw 찾기 페이지 스케치 해두기

## 작업 과정

> [!IMPORTANT]
>
> - 정확 : 메인 페이지의 설정창에서 '다크 모드'를 선택해도, 로그인 페이지나 수정 페이지로 이동하면 다시 '라이트 모드'로 돌아가는 현상이 있었습니다.
>         해결 하기위에 메인 페이지 로그인 페이지 회원 정보관리 페이지등의 자바 스크립트에 로컬 스토리지로 테마값을 확인하여 body에 .dark-mode 클래스를 주도록 하였음
>         각 페이지에 들어가는 스타일 시트의 다크모드 설정을 변경해줌
> - 정상원 : 사이드바 채팅 방 생성,새 채팅 버튼 추가, 로그인 후와 전 채팅 내역 분리(수정 중) 
> - 김영호 - 아이디 찾기와 비밀번호 찾기를 별도 페이지로 나누지 않고, 탭 버튼을 눌러 내용을 교체하는 방식으로 구현했다.
>           함수 하나로 버튼의 활성화 상태와 폼의 표시 여부를 동시에 제어하도록 짰다.
>           실제 백엔드 연동 전이라, 아이디 찾기 시엔 고정값(user1234)을, 비밀번호 찾기 시엔 랜덤 문자열을 생성해서 보여주도록 처리했다.


## 작업 회고

> [!IMPORTANT]
>
> - 정확 : 테마 설정을 유지되어야 하는 페이지가 어떤게 있을까 팀원들과 대화를 했고 로그인 페이지 메인페이지 결과 페이지 회원정보 수정 페이지에 테마 설정을 유지하고 로컬 스토리지를
>         이용 해야 하기에 각각의 페이지 마다 쓰이던 스타일 시트를 조금씩 추가하거나 변경하면서 개발자 도구를 이용해 최대한 시간을 단축 해보았고, 자바 스크립트는 인터넷 검색,
>         래퍼런스를 이용해보았는데 시간이 너무 오래걸렸다 아직 팀원들과 회의하면서 추가하고 만들어 보고싶은 부분들이 많은거같은데 남은시간이 얼마 없는거같아 걱정이됩니다.
> - 정상원 : 채팅방 만드는 작업이 별로 안 어려울줄았았는데 시간을 많이 썼다.
> - 김영호 : 본인인증 페이지의 팝업창(모달 창)으로 인증이 갔다는 메세지와 인증 메세지(아무 숫자 6자리)를 입력 시 인증이 완료되게 수정.
>           아이디/ 비밀번호 찾기 페이지의 기본적인 뼈대 완성.
